## Preliminary work on trajectories

Today I established a first reproducible baseline for working with the
Hanko back-trajectory data. The main goal was not yet to perform
concentration-based source analysis, but to understand the trajectory
data itself and produce diagnostic plots of air-mass history patterns.

I first confirmed the orientation of the trajectory arrays. The
trajectory latitude and longitude arrays have shape (30289, 97), where
each row is one trajectory and each column is a time step (hourly
?). By checking the distance of the first and last columns from the
Hanko station coordinates, I confirmed that the first column is the
arrival point at the station and the later columns move backwards in
time. Thus the trajectory order is Hanko → remote, with column 0
corresponding to arrival and the final column corresponding to the
farthest back-trajectory point.

I implemented a first gridding/counting approach using a fixed 1° × 1°
grid over the regional domain 50–75°N and 0–45°E (Northern Europe,
North Easter Europe and Western Russia). To match the earlier MATLAB
code prepared by a lab mate and the style of the reference
[paper](https://acp.copernicus.org/articles/13/2153/2013/acp-13-2153-2013.pdf),
trajectory coordinates are assigned to grid cells by rounding latitude
and longitude to the nearest full degree. For each trajectory, I count
**unique** visited grid cells, so a trajectory contributes at most one
count to a given cell (this agreed with the example code is
received). This is important because the diagnostic is intended to
count trajectory-cell visits rather than repeated time steps within
the same cell. **Note** that grid cells in Northern locations will be much 
smaller than further South. This means that they will naturally have fewer
visits which might introduce confounders.

I then pooled these trajectory-cell visit counts by calendar
month. This produced monthly trajectory count maps, which show the
absolute number of trajectories visiting each grid cell. These maps
are useful as denominator diagnostics: they show where later PSCF or
concentration-weighted maps will have strong or weak trajectory
support.

I also computed monthly trajectory-count anomaly maps. For each cell,
the anomaly is defined as:

```
monthly_count − annual_count/12
```

where the annual monthly mean is the total count over all months
divided by 12, cell by cell. These anomaly maps show seasonal
redistribution of trajectory histories. Positive values mean that a
cell is visited more often than usual in that month; negative values
mean that it is visited less often than usual. Importantly, negative
values do not mean that the cell is unimportant in absolute terms. A
nearby cell may still dominate the absolute trajectory counts while
appearing blue in a month where it dominates less than usual.

The anomaly maps required careful interpretation. A positive
near-station anomaly does not necessarily mean “more transport”;
because the back-trajectory window has fixed duration, it may instead
indicate more local residence, slower transport, or more recirculation
near Hanko. Conversely, negative near-station anomalies may indicate
less local residence and possibly faster or more long-range air-mass
histories. This suggests a useful future diagnostic: compute monthly
displacement distances after 24, 48, 72, and 96 timesteps (hours?) to
test whether autumn/winter trajectories tend to travel farther than
spring/summer trajectories.

I also improved plotting. The maps are now plotted using a Lambert
conformal conic projection, which is visually closer to the conic map
style used in the reference paper and other atmospheric trajectory
studies. For monthly anomaly panels, I use a shared color scale
centered at zero, so all months can be compared visually. This was an
important correction because independently scaled monthly plots were
misleading.

At this stage the repository contains a working baseline pipeline:

1. load cleaned trajectory arrays;
1. assign trajectory points to fixed rounded-degree grid cells;
1. count unique trajectory-cell visits;
1. pool counts by month;
1. compute monthly count anomalies;
1. plot monthly diagnostic maps.

These plots are **trajectory-density diagnostics only**. They **do not yet**
use particle concentration data and should not be interpreted as
source-contribution or concentration maps. The next major step is to
connect trajectories to concentration measurements and implement a
first high/low PSCF-style analysis.

## PSCF prototype work

Later in the day I moved from trajectory-density diagnostics to a first PSCF prototype. The main new task was to connect trajectory arrival times to DMPS concentration measurements. The DMPS matrix turned out to use paired rows: one row gives particle diameters `dp` for the size bins, and the following row gives the corresponding `dN/dlogDp` concentrations. The timestamp is stored in the first six columns as year, month, day, hour, minute, and second. I extracted one particle-size column as a first test case, split the matrix into diameter rows and concentration rows, constructed pandas timestamps, and filtered invalid or missing values.

For the first PSCF implementation, I defined high-concentration measurements using a percentile threshold. The current prototype uses the 75th percentile as the high-concentration cutoff. Trajectory times were rounded to seconds to remove small floating-point artifacts from MATLAB datenum conversion. I then identified high-concentration times and matched these to trajectory arrival times. For each grid cell, I counted all trajectories passing through the cell (`n_ij`) and the subset of high-concentration trajectories passing through the cell (`m_ij`). PSCF was then computed as `m_ij / n_ij`, with low-denominator cells masked.

In the current prototype, trajectory arrival times are matched to the most recent preceding concentration timestamp. In other words, for a trajectory arriving at time T, the associated concentration is taken from the latest available DMPS measurement with timestamp <= T. This is a simple first implementation and is useful for testing the end-to-end PSCF pipeline. However, since trajectory arrivals are hourly, **a more robust approach** would be to aggregate all DMPS measurements falling within the preceding hourly trajectory interval, for example (T - 1 h, T], and then classify the trajectory using the mean, maximum, or integrated concentration over that interval. This should reduce sensitivity to individual noisy measurements and better represent the concentration associated with each trajectory arrival.

The first monthly PSCF maps were successfully generated. The maps show coherent spatial structure rather than random noise, and higher PSCF values appear especially in eastern and southeastern transport sectors in several months. This is qualitatively similar to the type of regional signal seen in the reference paper, although the current plots are still preliminary. I wonder if the summer, particularly August, signal from the Russia/eastern sector might partly reflect wildfire smoke episodes, but this needs to be checked against external fire/smoke data and the actual high-concentration dates.
